Efficient Nearly-Fair Division with Capacity Constraints

This interactive demo runs the algorithm from Efficient Nearly-Fair Division with Capacity Constraints . Shoshan et al., AAMAS 2023 . Give it two agents, per-category quotas and item capacities, and it will compute an allocation that is EF[1,1] + Pareto-optimal whenever possible.

  1. Read the short algorithm overview below.
  2. Jump to the demo page, paste your own JSON, hit “Run”.
  3. Inspect the log to see every internal step.

Algorithm in a nutshell

For two agents:

  1. Category-wise W-max. Build a weighted bipartite graph in each category and use maximum-weight matching to maximise social welfare subject to quotas.
  2. Check if the resulting allocation is already EF[1,1]. If yes, we are done.
  3. If not, identify the envying pair – swap the best good/chore pair (same category) that reduces envy the most, and iterate.
